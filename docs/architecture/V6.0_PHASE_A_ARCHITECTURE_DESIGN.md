# V6.0 Visual Narrative System — Phase A Architecture Design

**版本**: Draft for Review  
**日期**: 2026-06-08  
**状态**: Phase A 已实现（见 `V6.0_PHASE_A_COMPLETION_REPORT.md`）  
**前置**: V5.2 Stable（`e80df53`）  
**ADR**: ADR-001 Experience Mode Architecture（延续，本阶段不修改 ADR 正文）

---

## 0. 文档目的

本文档定义 **V6.0 Phase A** 的架构边界、模块职责、数据模型与验收标准。

**本阶段不做 UI、不做 Dashboard、不做图片展示组件。**  
仅建立 Shared System 级 **Visual Asset Layer**，为世界提供可持久化、可缓存、可扩展的视觉资产注册与生成调度能力。

---

## 1. 背景与定位

### 1.1 V5.2 Stable 之后的核心闭环

```text
Character Brain
        ↓
Relationship Graph → Memory → Dynamics
        ↓
Objective System
        ↓
Plot Director
        ↓
Event Director → Director State Machine
        ↓
Prompt Runtime
        ↓
Story Output
```

V3.3 ~ V5.2 已完成 **Narrative Runtime Engine 第一阶段**。  
V5.2 不是功能节点，而是该阶段的 **Stable 版**。

### 1.2 V6.0 的战略定位

V6.0 不是「加几张图」，而是引入：

```text
Visual Narrative Architecture
```

即在叙事运行时之上，增加 **世界可视化资产层（Visual Asset Registry）**，使角色、地点、势力、关键场景成为可引用、可复用、可成长的世界事实。

### 1.3 与旧版 V6.0 草案的关系

仓库内 `doc/# PromptOS V6.0 设计文档.txt` 偏 **生成 + UI + Dashboard** 全栈。  
本 Phase A 设计 **收窄范围**，对齐当前冻结策略：

| 旧草案 | Phase A 本设计 |
|--------|----------------|
| `visual_assets.json` | **`visual_registry.json`**（统一注册表命名） |
| `ImageProvider` | **`VisualProvider`**（语义对齐 Narrative Visual） |
| `AgnesProvider` 散落 | **`AgnesVisualProvider`** 单一实现类 |
| `visual_manager.py` | 拆分为 **`asset_manager` + `visual_registry`** |
| 含 UI / Dashboard | **Phase A 禁止** |

全量 UI / 展示 / 动效留待 **V6.5+ / V6.6+**。

## 2. 冻结策略确认（开发前约束）

### 2.1 Stable 分支（已冻结）

| 模块 | 版本 |
|------|------|
| Character Brain | V3.3 Stable |

**仅允许**：Bug Fix、性能优化、测试补充、文档补充。

### 2.2 V6.0 开发隔离原则

- V6.0 代码 **新增** `engine/visual/`，**不修改** V5.2 Stable 模块的行为语义。
- 允许 **只读** 调用：`world_pack`、`memory`、`session_state`、`character_brain`（读角色描述）。
- 禁止在 V6.0 Phase A 中改动 `builder.py` 主 Prompt 链路（`visual_context` 可提供 API，但不强制注入叙事 Prompt，避免触及 Stable Prompt 预算）。

## 3. ADR-001 分类

### 3.1 Visual System = Shared System

| 资产类型 | 性质 | 分类 |
|----------|------|------|
| 角色立绘 | 世界角色事实 | Shared |
| 世界地图 | 世界地理事实 | Shared |
| 势力范围图 | 势力分布事实 | Shared |
| 关键场景图 | 剧情节点视觉记录 | Shared |

**禁止**：

```text
StoryPortrait / AdultPortrait
StoryMap / AdultMap
if experience_mode == "adult":  # 决定生成何种资产
```

体验差异（色调、裁剪、展示优先级）属于 **Mode Layer**（`VisualPriority` / `ImagePresentation`），**不得**分叉资产注册表或 Provider 实现。

### 3.2 Mode Layer 边界（Phase A 不实现）

| Mode Layer（V6.5+） | 说明 |
|----------------------|------|
| `VisualPriority` | 展示权重、主题色调 |
| `ImagePresentation` | 卡片/全屏/过渡 |
| `TransitionAnimation` | 场景切换动效 |
| Dashboard 可视化面板 | UI |

Phase A 只产出 **资产 URI + 元数据**；如何展示由后续 Phase 消费。

## 4. 目标架构

### 4.1 在 Narrative Stack 中的位置

```text
World Pack + Memory（事实源）
        ↓
Visual Registry（资产索引 · 本 Phase 核心）
        ↓
Asset Manager（生命周期 · 去重 · 持久化）
        ↓
VisualProvider（生成抽象 · AgnesVisualProvider 实现）
        ↓
Visual Cache（文件缓存 · 禁止重复请求）
        ↓
（未来）Presentation Layer · V6.5+
```

### 4.2 目录结构（Phase A）

```text
engine/visual/
├── __init__.py
├── visual_registry.py      # 注册表 CRUD、schema、ID 规范化
├── asset_manager.py        # 资产生命周期、save/load/reset 集成
├── visual_provider.py      # VisualProvider 抽象基类
├── agnes_visual_provider.py # AgnesVisualProvider（可 stub + 可选真实 API）
├── image_generation.py     # 生成编排：调 Provider、写 Cache、更新 Registry
├── visual_context.py       # 从 world/memory 构建生成 Prompt（Prompt Builder）
└── visual_cache.py         # 本地文件缓存、路径解析、去重键

data/
├── visual_registry.json    # 资产注册表（提交例外：!data/visual_registry.json 或 packaging default）

output/visual/              # 生成图片落盘（.gitignore，类比 chapter/output）
├── characters/
├── scenes/
├── maps/
└── factions/

packaging/defaults/
└── visual_registry.json    # 空 scaffold 出厂模板
```

**说明**：`image_generation.py` 负责「何时调 API」编排；`visual_context.py` 负责「生成什么 Prompt」；二者分离，避免业务层直接调 Agnes。

## 5. 核心模块设计

### 5.1 `visual_registry.py`

**职责**：Visual Asset Registry 唯一真源。

```python
# 概念 API（评审后冻结）
def load_registry() -> dict
def save_registry(registry: dict, *, persist: bool = True) -> None
def empty_registry() -> dict
def ensure_registry() -> dict

def get_asset(scope: str, asset_id: str) -> dict | None
def set_asset(scope: str, asset_id: str, record: dict) -> None
def list_assets(scope: str) -> dict

def normalize_asset_id(name: str) -> str  # "长公主" → "princess" 或 stable slug
```

**Scope 枚举（Phase A 固定）**：

| scope | 说明 |
|-------|------|
| `characters` | 角色立绘 |
| `locations` | 地点/区域（含世界地图锚点） |
| `factions` | 势力范围图 |
| `scenes` | 关键场景图 |

### 5.2 `asset_manager.py`

**职责**：对外统一入口（类比 `save_manager` 之于存档）。

```python
def get_or_request_character_portrait(name: str, world_pack: dict, *, turn: int = 0) -> dict
def get_or_request_scene_image(scene_key: str, context: dict, *, turn: int = 0) -> dict
def get_or_request_world_map(world_pack: dict, *, turn: int = 0) -> dict
def get_or_request_faction_map(faction_name: str, memory: dict, *, turn: int = 0) -> dict

def reset_visual_assets(*, persist: bool = True) -> None
```

**生命周期规则**：

1. 查 Registry → 有有效 `uri` 且文件存在 → **直接返回**（原则：同 ID 默认只生成一次）。
2. 无记录 → `visual_cache` 查去重键 → 仍无 → `image_generation` 调 Provider → 写盘 → 更新 Registry。
3. `updated_at` / `created_turn` / `provider` / `prompt_hash` 写入记录，供审计与防重复。

**与存档集成**（设计，Phase A 实现）：

| 操作 | 行为 |
|------|------|
| `save_manager.save` | snapshot 含 `visual_registry` + 可选 `visual_cache_index` |
| `save_manager.load` | 恢复 `visual_registry.json` |
| `reset_user_data` / New Story | `reset_visual_assets()` + 清空 `output/visual/` |
| `world.create_new_story` | 初始化空 registry 或从 default 复制 |

### 5.3 `visual_provider.py` — VisualProvider 抽象

**禁止**业务代码直接 `import agnes` 或散落 `OpenAIProvider` / `FluxProvider`。

```python
from abc import ABC, abstractmethod

class VisualProvider(ABC):
    @abstractmethod
    def generate_character(self, *, prompt: str, asset_id: str, size: str = "1024x1024") -> bytes: ...

    @abstractmethod
    def generate_scene(self, *, prompt: str, asset_id: str, size: str = "1024x1024") -> bytes: ...

    @abstractmethod
    def generate_world_map(self, *, prompt: str, asset_id: str, size: str = "1536x1024") -> bytes: ...

    @abstractmethod
    def generate_faction_map(self, *, prompt: str, asset_id: str, size: str = "1536x1024") -> bytes: ...

    @property
    @abstractmethod
    def provider_name(self) -> str: ...
```

**工厂**：

```python
def get_visual_provider() -> VisualProvider:
    # config.VISUAL_PROVIDER = "agnes" | "stub"
    ...
```

Phase A 必须提供 **`StubVisualProvider`**（写占位 PNG / 空文件路径），保证无 API Key 时测试与 CI 可通过；**`AgnesVisualProvider`** 实现同一接口。

### 5.4 `visual_context.py` — Prompt Builder

从 Shared System **只读** 拉取事实，构造生成 Prompt（不调用 LLM 叙事模型）。

| 方法 | 输入 | 输出 |
|------|------|------|
| `build_character_prompt` | world_pack.characters + brain 外貌摘要 | 英文/中文结构化 prompt |
| `build_scene_prompt` | session.scene + 最近 story 摘要 + location | scene prompt |
| `build_world_map_prompt` | world regions/locations | map prompt |
| `build_faction_map_prompt` | memory.factions + territories | faction map prompt |

**原则**（继承旧 V6.0 草案）：

- 图片是 **世界资产**，不是每回合产物。
- 场景图仅在 **关键节点**（如 Event Director `RESOLVED`、新 location 首次进入）触发——Phase A 可先暴露 API，**默认不挂接 run.py 自动触发**，避免改动 Stable 回合链。

### 5.5 `visual_cache.py`

```python
def cache_path(scope: str, asset_id: str, ext: str = "png") -> Path
def exists(scope: str, asset_id: str) -> bool
def write_bytes(scope: str, asset_id: str, data: bytes) -> Path
def prompt_hash(prompt: str) -> str  # 去重键
```

- 缓存目录：`output/visual/{scope}/{asset_id}.png`
- 同一 `(scope, asset_id)` 不得重复请求 Provider（除非 `force_regenerate=True` 显式 API，Phase A 可不暴露给 UI）。

### 5.6 `image_generation.py`

编排层：

```python
def generate_and_register(
    scope: str,
    asset_id: str,
    prompt: str,
    *,
    gen_fn: Callable[..., bytes],
    turn: int = 0,
) -> dict:
    """返回 registry record: { uri, provider, created_turn, updated_at, prompt_hash }"""
```

所有生成必须经过此模块，禁止绕过 Registry / Cache。

## 6. 数据模型 — `visual_registry.json`

### 6.1 顶层结构

```json
{
  "version": 1,
  "characters": {},
  "locations": {},
  "factions": {},
  "scenes": {}
}
```

### 6.2 单条资产记录（统一 schema）

```json
{
  "asset_id": "princess",
  "display_name": "长公主",
  "uri": "output/visual/characters/princess.png",
  "provider": "agnes",
  "kind": "portrait",
  "created_turn": 0,
  "updated_at": 1717843200,
  "prompt_hash": "a1b2c3...",
  "meta": {
    "width": 1024,
    "height": 1024
  }
}
```

### 6.3 ID 策略

- 优先使用 world_pack 中稳定 key（如角色 `id` 字段）；无则 `normalize_asset_id(中文名)` → ASCII slug。
- 同一 `display_name` 映射唯一 `asset_id`，避免重复立绘。

### 6.4 与 `MapSystem` / `FactionSystem` 关系

| 现有系统 | V6.0 关系 |
|----------|-----------|
| `world_pack` locations | `locations` scope 的 **事实源** |
| `memory.factions` | `factions` scope 的 **事实源** |
| `REGISTRY MapSystem` | 不替代；Visual 存 **图**，Map 存 **结构** |

## 8. Agnes 接入（Phase A 范围）

### 8.1 AgnesVisualProvider

- 实现 `VisualProvider` 四方法。
- API Key 从现有 `data/apikey.json` 扩展字段（如 `agnes_api_key`），**禁止提交 Git**。
- 网络失败 → 记录 `data/error.log`，返回可诊断错误；**不**崩溃叙事 `run.step()`（Visual 与叙事链解耦）。

### 8.2 默认四类生成

| 类型 | Provider 方法 | Registry scope |
|------|---------------|----------------|
| 人物立绘 | `generate_character` | `characters` |
| 关键场景 | `generate_scene` | `scenes` |
| 世界地图 | `generate_world_map` | `locations`（kind=world_map） |
| 势力范围图 | `generate_faction_map` | `factions` |
