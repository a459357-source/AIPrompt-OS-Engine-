## 架构要点

### 四源评分

```text
relationship_graph + relationship_memory + relationship_dynamics
        → midnight_talk / gift_exchange / confession / jealousy / …

objective_system (main/side progress)
        → clue_found / breakthrough / reveal_truth

plot_state (stage / progress / hooks / stall)
        → act_transition / boss_intro / major_twist

world_pack + memory factions + world_driver hooks
        → rebellion / war / assassination / political_pressure
```

### 输出示例

```json
{
  "event_id": "midnight_talk",
  "category": "relationship",
  "priority": 85,
  "reason": "trust_over_80",
  "participants": ["主角", "长公主"],
  "tags": ["bond", "private"]
}
```

### 冷却

- Catalog 内每事件 `cooldown`（回合）
- `data/event_history.json` 记录已推荐事件
- `is_on_cooldown()` 过滤重复推荐

### Prompt

- 块名：`{{DIRECTOR_PLAN}}`
- 预算：`EVENT_DIRECTOR_PLAN_MAX_CHARS ≤ 500`（≈ 300 tokens）
- 与 V5.1 关系块合计目标 ≤ 1700 tokens（V5.1 基线 1409 + Director ≤ 300）
