## 叙事循环（真实运行链路）

```mermaid
flowchart TD
    A[玩家选择 choice + 上轮 options] --> B[engine/run.py step]
    B --> C[build_prompt 组装 Prompt]
    C --> D[DeepSeek 生成 story + options]
    D --> E[apply_turn 写入 session]
    E --> F[_update_memory 角色记忆]
    F --> G[_update_relationship_graph]
    G --> H[relationship_update.apply_turn_relationship_updates]
    H --> I[Graph 六维 delta + Momentum 加成]
    I --> J[record_turn_memories 写 Memory]
    J --> K[apply_turn_dynamics]
    K --> L[Decay + Influence + Bond/Conflict + Triangle]
    L --> M[resolve_dynamics_events 写 Memory]
    M --> N[commit_runtime 原子落盘]
    N --> O[下一回合 build_prompt]
    O --> P[Brain / Objective / Plot / Event 只读召回]
    P --> C
```

## 分阶段代码锚点

### 1. 玩家行为 → Relationship Update

| 步骤 | 文件 | 函数 |
|---|---|---|
| 回合入口 | `engine/run.py` | `step()` → `_update_relationship_graph()` |

### 2. Memory Event

### 4. Brain / Objective / Plot Director 读取

| 消费者 | API | 注入位置 |
|---|---|---|

### 5. Prompt → 新剧情 → 再影响 Relationship

| 步骤 | 文件 | 说明 |
|---|---|---|

| AI 输出 options 含 metric 提示 | `prompt_template.yaml` | `affection+3 trust-1` 等 |
| 下回合回到步骤 1 | `relationship_update` | 解析 options 再次 delta |
