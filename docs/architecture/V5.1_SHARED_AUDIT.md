# V5.1 Review A — Shared System Audit

**日期**: 2026-06-08  
**范围**: Relationship Graph / Memory / Dynamics  
**ADR**: ADR-001 Experience Mode Architecture

---

## 审计结论

**结果**: PASS

三套关系子系统均为 **Shared System**，Story Mode 与 Adult Mode **读取同一份全局数据**，未发现体验模式分叉实现。

---

## 数据路径（全局唯一）

| 子系统 | 路径 | 模块 |
|---|---|---|
| Relationship Graph | `data/relationship_graph.json` | `relationship_core.py`, `relationship_update.py` |
| Relationship Memory | `data/relationship_memory.json` | `relationship_memory.py`, `relationship_recall.py` |
| Relationship Dynamics | `data/relationship_dynamics.json` | `relationship_dynamics.py`, `relationship_decay.py`, `relationship_influence.py`, `relationship_event_resolver.py` |

`config.py` 常量：`RELATIONSHIP_GRAPH_PATH`、`RELATIONSHIP_MEMORY_PATH`、`RELATIONSHIP_DYNAMICS_PATH` — 各一份，无模式后缀。

---

## 双实现排查

全仓库检索以下标识符：

```
StoryRelationshipGraph / AdultRelationshipGraph
StoryRelationshipMemory / AdultRelationshipMemory
StoryRelationshipDynamics / AdultRelationshipDynamics
```

**结果**: 零代码命中（仅 Phase B 完成报告中的合规声明）。

---

## 体验模式分支排查

对 `engine/relationship*.py` 检索 `adult`、`experience_mode`、`ADULT`、`Story Mode`：

**结果**: 零命中。关系引擎 **不读取** `config.get_experience_mode()` 或 `ADULT_MODE`。

Story / Adult 差异仅体现在：

- `builder.py` 选用不同 prompt 模板（`prompt_template.yaml` vs `prompt_template_adult_extreme.yaml`）
- 同一套 `{{RELATIONSHIP_*}}` 占位符，注入逻辑共用 `_relationship_*_context` 辅助函数
